**Step 3: Write a hand-written netlist matching what agents/simulate.md Stage A would produce for the fixture**

Create a temp working directory and netlist:

```bash
mkdir -p /tmp/librespin-07-02-e2e
cd /tmp/librespin-07-02-e2e
cat > circuit.cir <<'EOF'
Voltage Divider OP Analysis

Vin 1 0 DC 12
R1 1 2 10k
R2 2 0 3.3k

.op

.control
run
print v(2) I(Vin)
wrdata results.txt v(2)
quit
.endc

.end
EOF
```

**Step 4: Run ngspice and capture both streams**

```bash
cd /tmp/librespin-07-02-e2e
ngspice -b circuit.cir > sim-stdout.txt 2> sim-stderr.txt
EXIT=$?
cat sim-stdout.txt sim-stderr.txt > sim-combined.txt
ls -la circuit.cir results.txt sim-stdout.txt sim-stderr.txt 2>&1
echo "EXIT=$EXIT"
```

Capture: exit code, file sizes, full contents of sim-stdout.txt, sim-stderr.txt, and results.txt.

Verify:
- `results.txt` exists and is non-empty
- Second column of first data row ≈ 2.977 V (12 * 3.3 / 13.3)
- sim-combined.txt contains none of the 4 error substrings

```bash
ERRORS=""
grep -q "Timestep too small" sim-combined.txt && ERRORS="$ERRORS ERR-1"
grep -q "singular matrix" sim-combined.txt && ERRORS="$ERRORS ERR-2"
grep -q "Too many iterations without convergence" sim-combined.txt && ERRORS="$ERRORS ERR-3"
grep -q "Unable to find definition of model" sim-combined.txt && ERRORS="$ERRORS ERR-4"
echo "ERRORS=$ERRORS"
```

Capture the output — `ERRORS` should be empty on the happy path.

**Step 5: Exercise at least ONE error pattern for real**

Create `circuit-broken.cir` with a deliberate singular-matrix fault (floating node):

```bash
cat > circuit-broken.cir <<'EOF'
Broken Voltage Divider

Vin 1 0 DC 12
R1 1 2 10k
* R2 intentionally missing — node 2 is floating

.op

.control
run
wrdata results-broken.txt v(2)
quit
.endc

.end
EOF

ngspice -b circuit-broken.cir > broken-stdout.txt 2> broken-stderr.txt
cat broken-stdout.txt broken-stderr.txt > broken-combined.txt
grep -q "singular matrix" broken-combined.txt && echo "ERR-2 MATCHED" || echo "ERR-2 NOT MATCHED — INVESTIGATE"
```

Capture the broken run output. If the error substring differs from what 07-RESEARCH.md documented, RECORD THE DIFFERENCE — this is exactly the kind of real-world fact the plan-01 skill needs to reflect. If necessary, update `agents/simulate.md` with the corrected substring and note the fix in the evidence file.
