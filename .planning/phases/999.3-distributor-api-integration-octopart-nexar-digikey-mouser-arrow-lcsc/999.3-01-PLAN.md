**Step 3: Define helper functions** (skill instructs Claude to define these in a Bash block used throughout):
```bash
read_credential() {
  local section="$1" key="$2" file="$HOME/.librespin/credentials"
  grep -A50 "^\[$section\]" "$file" | grep -m1 "^$key" | awk -F' = ' '{print $2}' | tr -d '\r\n'
}

write_credential() {
  local section="$1" key="$2" value="$3" file="$HOME/.librespin/credentials"
  # Update key within section â€” GNU sed required (Linux primary)
  sed -i "/^\[$section\]/,/^\[/{s|^$key = .*|$key = $value|}" "$file"
}
```

**Nexar:**
- Prompt: client_id, client_secret
- Write to `[nexar]` section: `client_id`, `client_secret`
- Note to show user: "Nexar free tier: 100 matched parts total (not monthly). Parts used will be tracked in credentials file."
- Token acquisition (write result to credentials):
```bash
NEXAR_RESPONSE=$(curl -s --request POST 'https://identity.nexar.com/connect/token' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'grant_type=client_credentials' \
  --data-urlencode "client_id=$NEXAR_CLIENT_ID" \
  --data-urlencode "client_secret=$NEXAR_CLIENT_SECRET" \
  --data-urlencode 'scope=supply.domain')
NEXAR_TOKEN=$(echo "$NEXAR_RESPONSE" | jq -r '.access_token // empty')
# Calculate expiry: 24 hours from now
NEXAR_EXPIRY=$(date -u -d "+23 hours" +%Y-%m-%dT%H:%M:%SZ)
```
- Validation test:
```bash
NEXAR_TEST=$(curl -s -X POST 'https://api.nexar.com/graphql/' \
  -H "Authorization: Bearer $NEXAR_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"query": "query { supSearchMpn(q: \"LM358\", limit: 1) { hits } }"}')
NEXAR_HITS=$(echo "$NEXAR_TEST" | jq -r '.data.supSearchMpn.hits // 0')
# VALID if NEXAR_HITS > 0
```
- On success: write `access_token`, `token_expires`, reset `parts_used = 0`

**DigiKey:**
- Prompt: client_id, client_secret
- Note: "Obtain credentials at developer.digikey.com â€” create a Production app to get client credentials grant access."
- Token acquisition:
```bash
DK_RESPONSE=$(curl -s -X POST 'https://api.digikey.com/v1/oauth2/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d "client_id=$DK_CLIENT_ID&client_secret=$DK_CLIENT_SECRET&grant_type=client_credentials")
DK_TOKEN=$(echo "$DK_RESPONSE" | jq -r '.access_token // empty')
# Expiry: 9 minutes from now (599s - 60s buffer)

```
- Validation test:
```bash
DK_TEST=$(curl -s -X GET "https://api.digikey.com/products/v4/search/LM358/productdetails" \
  -H "Authorization: Bearer $DK_TOKEN" \
  -H "X-DIGIKEY-Client-Id: $DK_CLIENT_ID" \
  -H "X-DIGIKEY-Locale-Site: US" \
  -H "X-DIGIKEY-Locale-Language: en" \
  -H "X-DIGIKEY-Locale-Currency: USD")
# VALID if HTTP response does not contain "error" or "unauthorized" and contains "Product"
DK_VALID=$(echo "$DK_TEST" | jq -r 'if .Product then "yes" else "no" end')
```
- CRITICAL: Always include both `Authorization: Bearer TOKEN` AND `X-DIGIKEY-Client-Id: CLIENT_ID` headers on every request.
- On success: write `access_token`, `token_expires`
