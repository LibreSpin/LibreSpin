**Phase-by-phase expected behavior:**

**Phase 2 — Architecture Drafting (CW-03):**
- State dispatches to Phase 2 from `3-requirements-gathering`
- Agent reads `.librespin/01-requirements/requirements.yaml` and `config.yaml`
- Agent generates `draft_count` (default 5) diverse architecture concepts
- Writes concept-*.md files to `.librespin/02-concepts/`
- Writes overview.md to `.librespin/02-concepts/`
- Updates state.md to `phase: '2-architecture-drafting'`
- If config.yaml missing or draft_count invalid: blocking bug — fix per Pitfall 3

**Phase 3 — Validation Gate (CW-04):**
- State dispatches to Phase 2.5 + Phase 3 from `2-architecture-drafting`
- Agent validates concepts against confidence_threshold (default 80)
- Uses WebSearch for component feasibility research
- Adds validation sections to concept files
- Writes validation summary to `.librespin/03-validation/`
- Updates state.md to `phase: '3-validation-gate'`

**Phase 4 — Component Research (CW-05):**
- State dispatches to Phase 4 from `3-validation-gate`
- Agent researches specific components for validated concepts
- Writes BOM files with real MPNs to `.librespin/04-bom/bom-{name}.md`
- Updates state.md to `phase: '4-component-research'`

**Phase 5 — Concept Generation (CW-06):**
- State dispatches to Phase 5 from `4-component-research`
- Agent generates block diagrams and spec analysis
- Writes analysis files to `.librespin/05-detailed-designs/analysis-{name}.md`
- Updates state.md to `phase: '5-concept-generation'`

**Phase 6 — Self-Critique & Refinement (CW-07):**
- State dispatches to Phase 6 from `5-concept-generation`
- Agent performs iterative self-critique (up to iteration_limit passes)
- Writes score files to `.librespin/06-refinement/score-{name}.md`
- Writes refinement-log.md to `.librespin/06-refinement/`
- Updates state.md to `phase: '6-self-critique'`
- Monitor for context pressure (Pitfall 5) — note if agent truncates or skips steps

**Phase 7 — Final Output (CW-08):**
- State dispatches to Phase 7 from `6-self-critique`
- Agent discovers concepts from Phase 6 score files
- Generates comparison matrix and recommendation
- Writes to `.librespin/07-final-output/`
- Updates state.md to `phase: '7-final-output'`

**Bug fix protocol (per D-05):**
If any phase fails or produces incorrect/missing output:
1. Identify the root cause in SKILL.md
2. Fix the blocking issue (minimum viable fix)
3. Re-run the failed phase
4. Document the bug and fix in the checklist's "Bug Fixes Applied" table

**Do NOT fix (per D-06):**
- Prompt quality issues (e.g., concepts are too similar, block diagrams are ugly)
- Output formatting issues (e.g., markdown rendering, column alignment)
- Context optimization (even if pressure observed — just document it)

**After all phases complete:**
Update the "Context Pressure Observations" section in the checklist with:
- The phase where context pressure was first observed (or "none")
- Symptoms observed (or "none")
- Recommendation for OPT-01 priority (low/medium/high based on observation)
